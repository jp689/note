# 项目结构与功能说明文档

> 项目：**檐雨成谱 — 国风建筑气候适应交互实验**
> 类型：教育向 Web 交互游戏 / 建筑科普参数模拟器
> 整理用途：帮助后续 Agent、开发者或评委快速理解项目架构、功能模块、资源组织与部署方式。

## 一、项目基本介绍

### 1.1 项目名称

**檐雨成谱 — 国风建筑气候适应交互实验**

### 1.2 项目类型

教育向 Web 交互游戏 / 建筑科普参数模拟器。

### 1.3 项目核心功能

项目以中国传统屋顶形制（当前主线为**硬山顶**）与气候适应智慧为主题，玩家以“误入旧木场的学徒”身份，在老匠人引导下经历：

```text
观察 → 选材 → 拼图 → 三维升维 → 视频讲解 → 风雨推演 → AI 结算批语
```

通过雨雪粒子物理模拟、参数调节（坡度 / 出檐 / 材质 / 天气）、手势识别、自适应音乐生成等手段，将建筑知识转化为可操作、可验证的交互体验。

### 1.4 项目面向用户

- 计算机设计大赛评委与观众
- 对中国传统建筑文化感兴趣的大众用户
- 建筑学 / 土木工程专业学生（辅助教学）
- 小程序 / Web 端科普平台用户

### 1.5 主要交互流程

1. **启卷（开始页）**
   自动滚动的国风背景 + 标题动画 → 点击“启卷”按钮。

2. **硬山顶图谱观察**
   轮播硬山顶实拍图，自动缓移 → 点击“进入选材”。

3. **构件选材**
   从 7 种几何图形卡片中选 4 个 → 点击“进入拼图”。

4. **二维拼构**
   拖拽 / 手势捏合，将坡面与山墙拼回展开图 → 全部归位后点击“升维成顶”。

5. **三维屋顶预览**
   Three.js 加载 GLB 模型，支持手势旋转 / 缩放 → 点击“观看结构讲解”。

6. **视频讲解**
   播放硬山顶讲解 MP4 → 点击“进入风雨推演”。

7. **风雨推演（主实验）**
   调节出檐 / 坡度 / 材质 / 天气 → 点击“落雨推演” → 粒子模拟 → 点击“收势”。

8. **AI 结算**
   统计数据 + AI 批语（老匠人口吻） → “再试一次” 或 “去实地探究”。

9. **屋脊寻踪（探索模式）**
   iframe 嵌入高德地图，点击不同屋顶形制点位。

### 1.6 项目整体完成度判断

整体完成度约 **70% - 80%**。

主线硬山顶入门课（序章）功能完整，包含较完整的学习流程。但目前仍存在以下未完成项：

- 探索模式中歇山顶、庑殿顶仅地图锁定展示，未开放互动关卡。
- 故事模式当前仅含一个关卡（`prologue`），`STORY_LEVELS` 数组仅含一项。
- 部分 `explore/` 目录下的悬山顶关卡实现是独立子页面，尚未完全整合进主流程。

## 二、技术栈说明

### 2.1 前端框架 / 构建工具

- **无框架**：原生 HTML + ES Module + Canvas 2D。
- 主项目不使用 React / Vue。
- 通过 `importmap` 映射 Three.js，使用原生 ES 模块组织。
- **无构建工具**：无 Vite / Webpack 配置。
- 通过 `<script type="module">` 直接加载。
- 可直接部署为静态站点，但本地建议使用 HTTP 服务器运行，避免 CORS 问题。

### 2.2 使用到的主要库

| 库 | 版本 | 用途 | 加载方式 |
|---|---:|---|---|
| Three.js | r160 | 三维屋顶模型预览、GLB 模型加载 | 本地 `vendor` 目录 |
| GLTFLoader | r160 配套 | 加载 `.glb` 三维模型 | 本地 `vendor` 目录 |
| MediaPipe Hands | 0.4.x | 摄像头手势识别（拼图、3D 预览操控） | CDN |
| MediaPipe Camera Utils | 0.3.x | 摄像头采集 | CDN |
| MediaPipe Drawing Utils | 0.3.x | 手部关键点绘制 | CDN |
| 高德地图 JS API | - | 探索模式地图展示 | CDN（通过 `loader.js`） |
| Google Fonts | - | Ma Shan Zheng、ZCOOL XiaoWei 等国风字体 | Google Fonts CDN |

### 2.3 特殊能力使用情况

| 能力 | 是否使用 | 说明 |
|---|---|---|
| Three.js / WebGL | 是 | `src/ui/roof-preview-3d.js` 加载 GLB 模型 |
| Canvas 2D | 是 | 主要渲染引擎，雨雪粒子模拟全在 Canvas 2D 上 |
| 音频 | 是 | 天气音效、BGM（8 首阶段音乐）、自适应五声音阶生成 |
| 摄像头 | 是 | MediaPipe 手势识别、颜色追踪（风向）、运动检测 |
| 麦克风 | 是 | 声音强度映射雨量 |
| AI API | 是 | 通义千问（Qwen）云函数，用于生成老匠人结算批语 |
| 手势识别 | 是 | MediaPipe Hands，用于拼图抓取和 3D 预览操控 |

### 2.4 后端或接口情况

- 腾讯云函数 SCF：`https://1320505213-i96el273o0.ap-shanghai.tencentscf.com`
- 模型：`qwen3.6-plus`
- 仅用于结算时生成 AI 批语，不影响核心玩法。
- 请求为 `fetch POST`，携带 `system/user prompt` 和参数上下文。

### 2.5 部署方式或运行方式

- 纯静态前端项目。
- 可通过本地 HTTP 服务器运行。
- 无需 Node.js 构建步骤。
- 无 SSR。

### 4.9 AI 结算批语模块

| 项目 | 内容 |
|---|---|
| 相关文件 | `src/ai/ai-orchestrator.js`（5KB） |
| 主要功能 | 调用腾讯云函数 Qwen API，生成老匠人口吻的结算批语 |
| API 地址 | `https://1320505213-i96el273o0.ap-shanghai.tencentscf.com` |
| 模型 | `qwen3.6-plus` |
| 请求方式 | `POST`，`Content-Type: application/json` |
| 请求参数 | `{ model, messages[{ role, content }], stream: false, temperature: 0.55 }` |
| 返回结构 | `payload.choices[0].message.content`（字符串） |
| Prompt 模板 | system prompt 定义老匠人口吻规则；user prompt 填充关卡数据、参数、统计 |
| 失败处理 | `try-catch` 捕获，失败返回 `null`，UI 不显示 AI 批语 |
| 注意事项 | API endpoint 和模型名硬编码在源码中，无环境变量 |

## 九、目前可能存在的问题

### 9.1 未完成的功能

- `explore/app.js` 中有独立的悬山顶游戏逻辑，与主项目 `src/` 中的逻辑有部分重复。
- `explore/images/back.png` 与 `images/back.png` 可能是同一文件的重复。

- `image/` 与 `images/` 两个目录并存，容易混淆。
- `硬山顶1.glb` 放在 `image/` 目录，其他资源在 `images/`。
- 中文文件名（如 `images/硬山顶/`）可能在某些服务器上有编码问题。

### 9.4 潜在死代码

- `video-player.js`：旧版视频播放器，已被 `roof-video-stage.js` 替代。
- `debug-*.js` 文件已在 git 中标记删除，但 `.gitignore` 也有规则。

### 9.5 资源路径问题

- `image/硬山顶1.glb` 路径使用 URL 编码 `%E7%A1%AC%E5%B1%B1%E9%A1%B61.glb`，需确保服务器正确处理。
- 中文目录名 `images/硬山顶/` 在某些环境下可能有问题。

### 9.6 API Key 暴露风险

- 高德地图 Key 和安全码硬编码在 `explore/app.js`。
- 腾讯云函数 endpoint 硬编码在 `src/ai/ai-orchestrator.js`。
- 建议通过环境变量或后端代理保护。

### 9.7 CORS 问题

- 直接双击 HTML 文件时，ES Module 加载会因 CORS 失败。
- 需要 HTTP 服务器环境。

### 9.8 大文件问题

- `image/硬山顶1.glb`（91MB）：首次加载极慢。
- `images/back.png`（10MB）：开始页背景较大。
- `images/侧栏图/left.png`（9MB）、`right.png`（10MB）：侧栏装饰图较大。
- 未见明显的图片压缩或懒加载策略。

### 9.9 移动端适配

- 移动端横竖屏布局适配。

## 十、后续优化建议

### 10.1 架构层面

- 将 `image/` 与 `images/` 统一为一个资源目录，减少路径混乱。
- 将探索模式中的悬山顶逻辑整合进主项目关卡系统。
- 清理旧版或重复文件，例如 `video-player.js`。
- 将大体积资源迁移至 CDN 或对象存储，并开启压缩与缓存。

### 10.2 安全层面

- 不要在前端硬编码 API Key、安全码、模型接口地址。
- 高德地图 Key 建议配置白名单和 Referer 限制。
- Qwen / 腾讯云函数接口建议增加鉴权、防刷、速率限制。
- 将 AI 调用封装到后端代理层，避免接口被直接调用。

### 10.3 性能层面

- 压缩大图资源，例如 `back.png`、侧栏图。
- GLB 模型可考虑 Draco 压缩或拆分加载。
- 对视频、BGM、模型做懒加载。
- 首屏只加载首页必要资源，进入阶段后再加载对应资源。

### 10.4 功能层面

- 补全歇山顶、庑殿顶、悬山顶等后续关卡。
- 增加关卡进度保存功能。
- 增加移动端触控优化。

## 十二、快速结论

这个项目是一个以中国传统屋顶为主题的**国风建筑科普交互游戏**。目前已经具备完整的硬山顶主线体验，包括观察、拼图、三维预览、视频讲解、风雨模拟和 AI 结算。项目亮点在于视觉风格鲜明、交互链路完整、融合了 Canvas 物理模拟、Three.js、MediaPipe、Web Audio 与 AI 批语等多种能力。

目前主要短板集中在：

- 后续屋顶关卡未完全开放。
- 探索模式与主流程整合度不够。
- 大文件资源较多，加载性能存在压力。
- API Key / endpoint 暴露在前端。
- 移动端适配和资源加载优化仍需加强。

整体来看，项目已有较强展示效果，适合作为比赛作品继续打磨。后续重点应放在**性能优化、关卡补全、接口安全、移动端适配、资源整理**五个方面。
